# HYPERPLASTICITY PROTOCOL V5

HPP V5 is a fresh-start post-LLM architecture and local-first neuroplasticity operating system.

It is being rebuilt from the original HPP field notes into two cleanly separated lives:

- a buyer-facing IP asset for explaining and demonstrating the architecture
- a private operating core for state training, shop execution, field work, and future robotics

## Core Promise

Train the state. Rewrite the loop. Become the signal.

## Current App

The first V5 app is a React/TypeScript local cockpit with:

- state check-ins
- protocol selection
- step-by-step loop execution
- after-state/reflection capture
- local evidence logging
- Habit-14 progress
- Battery, Plugged, and Demo power modes

Run locally:

```powershell
npm install
npm run dev
```

## Initial Product Shape

- A daily command center for state, protocol, and next action.
- A protocol library for repeatable training loops.
- A journal that blends freeform reflection with structured signals.
- A pattern layer that shows what changes the user over time.
- An AI guide that helps interpret logs, design experiments, and adapt protocols.

## Source Inheritance

V5 can borrow from the Aural-Nexus portfolio, especially the original `08_Hyperplasticity_Protocol`, but the project should preserve the core hypothesis first:

- Repetition creates plastic change.
- Fourteen successful loops can mark stabilization.
- Nurture and stress need different protocol paths.
- The system must preserve evidence of what works.

The previous version also proved several model-side directions worth carrying forward:

- recurrent depth through shared weights
- Habit-14 stabilization gates
- microglia-style pruning/protection telemetry
- nurture versus sentinel routing
- staged developmental curriculum
- explicit CUDA device selection

## Field Lab Rule

HPP is being designed for real mobile constraints: laptop, RTX 4050-class GPU, battery limits, intermittent connectivity, and stop/resume work. Heavy processing should be gated behind plugged-in mode and explicit CUDA checks.
