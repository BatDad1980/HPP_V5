# Product Ideas

## Main Screens

### Today

The first screen. It shows current state, active identity, selected protocol, quick check-in, and next action.

### Protocols

A library and editor for repeatable training loops.

Each protocol can include:

- Intent
- Trigger
- Duration
- Steps
- Contraindications or cautions
- Before/after signals
- Reflection prompt
- Tags

### Journal

A reflection space that supports both freeform text and structured metadata.

Entry types:

- Check-in
- Protocol reflection
- Breakthrough
- Resistance
- Dream or symbol
- Experiment note
- Weekly review

### Patterns

Charts, streaks, correlations, and AI summaries.

The point is not gamification. The point is evidence.

### Architect

The deeper layer where the user defines identities, values, quests, constraints, and active transformations.

## Example Protocols

### Nervous System Reset

Intent: reduce activation and return to baseline.

Steps:

1. Breathe low and slow for two minutes.
2. Relax jaw, shoulders, and hands.
3. Name three sensations without interpreting them.
4. Choose the next smallest useful action.

### Focus Ignition

Intent: enter a work sprint with less friction.

Steps:

1. Choose one target.
2. Remove one source of noise.
3. Set a short timer.
4. Begin with the easiest visible action.

### Identity Rehearsal

Intent: practice acting from a chosen identity.

Steps:

1. Name the identity.
2. Name the current resistance.
3. Ask: what would this identity do in the next five minutes?
4. Do that action before negotiating.

## Early AI Features

- Summarize the last seven days.
- Suggest one protocol adjustment.
- Convert a journal entry into signals and tags.
- Identify repeated resistance patterns.
- Generate a weekly experiment.
