# Tap-Out Boundary Sweep Summary

This sweep increases extreme out-of-distribution noise to find where each tolerance profile taps out.

## Setup

- Mode: `plugged`
- Device: `NVIDIA GeForce RTX 4050 Laptop GPU`
- Profiles: `low-tolerance, standard, high-intensity`
- Seeds: `14, 21, 42`
- Noise scales: `0.8, 1.2, 1.6, 2.0, 2.4, 2.8, 3.2, 3.6, 4.0`

## Boundary Table

| Profile | First tap-out noise | First zero-failure noise | Max no-tapout noise | Final tap-out mean | Final tap-out failure mean |
| :--- | ---: | ---: | ---: | ---: | ---: |
| low-tolerance | 2.0 | 0.8 | 1.6 | 0.2 | 0.0 |
| standard | 2.0 | 0.8 | 1.6 | 0.2 | 0.0 |
| high-intensity | 2.8 | 0.8 | 2.4 | 0.2 | 0.0 |

## Interpretation

The sweep separates intensity tolerance from safety. A profile can tolerate more unfamiliar noise before tapping out, but if it never taps out it can preserve failures at higher noise levels.

## Boundary

This is synthetic mechanism evidence. Noise is generated by the harness, and the OOD estimator is hand-built.
