# Roadmap

## Phase 0: Foundation

- Define product vision.
- Preserve original HPP hypothesis and source inheritance map.
- Choose stack.
- Create project scaffold.
- Establish app layout and visual direction.
- Define initial data models.

## Phase 1: Local Core

- Today screen.
- State check-in.
- Protocol library.
- Protocol run flow.
- Habit-14 stabilization tracking.
- Nurture mode and Sentinel mode interaction paths.
- Local persistence.
- Battery-safe and plugged-in power modes.
- Clear separation between buyer demo data and private field-ops data.

## Phase 2: Reflection Loop

- Journal entries.
- Protocol reflections.
- Before/after state comparison.
- Weekly review screen.

## Phase 3: Pattern Layer

- Basic charts.
- Trends by signal.
- Protocol effectiveness summaries.
- Tag and identity filters.

## Phase 4: AI Guide

- AI summary of recent entries.
- Protocol suggestions.
- Experiment design.
- Identity-aware coaching prompts.
- CPU/GPU execution mode selection.
- CUDA diagnostics and run-budget reporting.
- Optional speech-adapter research track with transcript logging, repetition metrics, and buyer-safe evidence boundaries.

## Phase 5: System Maturity

- Import/export.
- Backups.
- Search.
- Custom metrics.
- Advanced protocol templates.
- Optional cloud sync.
- Buyer packet export.
- Private operating-core workspace.
- Robotics/shop integration adapters.

## Suggested Initial Stack

- React
- TypeScript
- Vite
- Tailwind CSS or carefully structured CSS
- SQLite through a local API, or JSON persistence for the first prototype

## First Milestone

Build a clickable local app where the user can:

1. Complete a current-state check-in.
2. Select a protocol.
3. Run through protocol steps.
4. Write a short reflection.
5. See the result saved locally.
