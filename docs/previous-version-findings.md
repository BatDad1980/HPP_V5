## Strong Inheritance Candidates

### Recurrent Depth Instead Of Static Scale

The `HyperPlasticCore` uses a shared transformer workshop repeatedly rather than adding a large stack of unique layers. This is the central post-LLM efficiency idea to preserve and sharpen.

### Habit-14 Stabilization

The first HPP uses fourteen successful loops as a stabilization gate. V5 should keep Habit-14 as both a model-side concept and a user/product mechanic.

### Microglia-Style Filtering

The `KarmicMicrogliaFilter` tracks noisy versus stable pathways through karma and vairagya buffers. V5 can translate this into:

- model telemetry
- protocol evidence scoring
- buyer-facing explanation of adaptive pruning/protection

### Nurture And Sentinel Routing

The Guardian ecosystem separates slow reflective learning from fast protective routing under stress. V5 should keep this distinction as a core product and architecture principle.

### Developmental Curriculum

The previous build stages infant, toddler, preschool, school, and adolescent cores. V5 can retain this as either:

- model maturity levels
- protocol maturity levels
- buyer-facing roadmap phases

### GPU-Aware Inference

The previous inference engine already selects `torch.device("cuda" if torch.cuda.is_available() else "cpu")` and moves model pieces onto the selected device. V5 should make this more explicit, with battery-aware modes and device telemetry.
