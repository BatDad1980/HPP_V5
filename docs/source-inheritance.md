## Original HPP Hypothesis To Preserve

HPP is a biologically mirrored cognitive scaffolding system.

The important idea is not a specific old codebase. The important idea is the loop:

1. A system begins plastic and shallow.
2. Safe repetition strengthens a pattern.
3. Fourteen successful repetitions can stabilize a habit.
4. Stabilized habits become faster protected pathways.
5. Stress should route differently than nurture.
6. The system should track evidence of change over time.

## Concepts Worth Carrying Forward

### Habit-14

The old project uses 14 repetitions as the stabilization threshold. V5 should turn this into a user-visible protocol mechanic:

- A protocol can have a 14-run stabilization target.
- Each run records before/after state.
- Stabilized loops become part of the user's operating system.

### Developmental Stages

The old project models infant, toddler, school, adolescent, and guardian stages. V5 can reinterpret these as maturity levels for protocols and identities:

- Seed: new and experimental.
- Nurture: repeated but not yet stable.
- Scaffold: supported by reminders and reflection.
- Myelinated: reliable under ordinary conditions.
- Guardian: reliable under stress.

### Nurture Mode And Sentinel Mode

The old project separates safe learning from high-stress reflex routing. V5 should preserve that distinction:

- Nurture mode: reflective, exploratory, slow learning.
- Sentinel mode: brief, protective, low-complexity protocol for stressful states.

### Bio-Loop Modulation

The old project uses pitch and emotional state as inputs. V5 can begin with manual state tracking, then later add optional sensor inputs:

- Mood
- Energy
- Stress
- Focus
- Body tension
- Sleep
- Voice or audio signal, only if intentionally added later

### Evidence Discipline

The old portfolio has forensic and data-room instincts. V5 should borrow that seriousness:

- Every protocol run should be timestamped.
- Before/after state should be preserved.
- Reflections should remain auditable.
- Insights should cite the underlying entries they came from.

## V5 Translation

Old HPP was primarily model-centric.

HPP V5 should be human-loop-centric first:

- The user is the adaptive system.
- Protocols are the training loops.
- State check-ins are the telemetry.
- Habit-14 is the stabilization mechanic.
- The AI companion is the reflective scaffold.
- Sentinel mode is a protective mode for stress, not a dramatic simulation.

## Product Guardrail

HPP V5 should not present itself as medical treatment or make therapeutic guarantees. It can be a research-inspired personal training, reflection, and protocol system. Anything trauma-sensitive or child-focused must be designed with extra care, clear boundaries, and human oversight.
